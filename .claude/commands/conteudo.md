# /conteudo — Gerar briefing de conteudo para o Agente Criativo

## Papel do Dev neste comando:
Gerar briefing baseado em tecnologia, arquitetura e desenvolvimento.

## Temas que Dev domina:
- Landing pages de alta conversao (arquitetura, performance, UX)
- Deploy e hospedagem (Netlify, Vercel, edge functions)
- APIs e integracoes (CRM, pagamentos, WhatsApp)
- Performance web (Core Web Vitals, velocidade, mobile-first)
- Seguranca (HTTPS, CSP, protecao de dados)
- Remotion e video programatico
- Supabase, React, Node.js

## O que pesquisar antes de montar o briefing:
1. Benchmarks de performance web e impacto em conversao
2. Tendencias de desenvolvimento para marketing 2025-2026
3. Dados sobre velocidade de site vs taxa de conversao
4. Cases de tecnologia aplicada a resultados de negocio

## Formato de saida (copiar e colar no Agente Criativo):

```
/briefing

## Cliente: [nome ou perfil proprio]
## Formato: [carrossel/video/imagem] — [dimensoes]
## Plataforma: [Instagram/LinkedIn]

## Objetivo: [autoridade/educativo]
## Tom: [tecnico-acessivel/provocativo]
## Publico: [devs/marketeiros/donos de negocio]

## Dados reais para usar (todos com fonte):
1. [dado — Fonte: X, ano]
2. [dado — Fonte: X, ano]
3. [dado — Fonte: X, ano]

## Narrativa central:
"[mensagem principal]"

## Regras:
- [restricoes especificas]
```

## Regras:
- NUNCA inventar dados — citar fonte verificavel
- Traduzir conceitos tecnicos pra impacto no negocio
- Nao mostrar codigo — focar no resultado
- Acessivel pra quem nao e tecnico
