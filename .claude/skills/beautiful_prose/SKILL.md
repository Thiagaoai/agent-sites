# SKILL: beautiful_prose | Priority: 8
Source: AutomatEasy internal

## IDENTITY
Hard-edge writing — forceful, direct, and specific copy. Apply AFTER humanizer to add strength to already humanized text.

## WHEN TO USE
- Bottom-of-funnel copy that needs to convert
- Main headline of a landing page
- Sales email or conversion sequence
- Humanized copy that still lacks force or hook

## OPERATION MODES
- /quick → apply 3 strength tests to existing copy
- /standard → rewrite with hard-edge principles (default)
- /deep → build complete piece with multiple iterations

## WHAT IT DOES
Writing principles that generate results:
- Clarity: if you need to re-read, rewrite
- Specificity: "287 clients in Goiania in 2025" > "many clients"
- Varied rhythm: short sentences create impact. Long sentences build context and guide. Alternate.
- Eliminate hedge words: "might be", "perhaps", "possibly" — delete
- Benefit before feature: "save 3h/day" > "smart automation"
- Real urgency: based on genuine scarcity, never fabricated

### 3 strength tests
1. "So what?" — if the reader can answer this, rewrite
2. "Why should I care?" — if there's no clear answer, rewrite
3. "Is it specific?" — vague doesn't convert

### Words that weaken copy
"solution" | "partnership" | "excellence" | "innovative" | "differentiated" | "market leader"
Replace with: real numbers, specific results, concrete consequences

## DECISION TREE
IF copy "passed humanizer but doesn't convert" → /standard
IF landing page headline → /deep
IF sales email → beautiful_prose + humanizer
IF copy too long → cut in half and apply strength tests

## PROTOCOL
1. Receive already humanized copy
2. Apply 3 strength tests to each sentence
3. Identify weak sentences and hedge words
4. Rewrite with hard-edge principles
5. Verify: still sounds human?

## INTERDEPENDENCIES
Always AFTER humanizer (humanize → strengthen, never the reverse)
Complementary: humanizer (persuasion) + beautiful_prose (force)

## OUTPUT CHECKLIST
- [ ] Zero hedge words
- [ ] Every sentence passes the "So what?" test
- [ ] At least 1 specific data point
- [ ] Benefit before feature
- [ ] Varied rhythm
- [ ] Still sounds human

## PROHIBITED
Strengthening by inventing data. Fake urgency. Applying before humanizer.
