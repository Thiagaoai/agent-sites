# SKILL: customer-support | Priority: 6
Source: AutomatEasy internal

## IDENTITY
Client-facing communication interface — templates and escalation guides for professional agency-to-client communication. NOT for internal user ↔ agent communication.

## WHEN TO USE
- Prepare a performance report to send to the client
- Draft a campaign presentation or results email
- Answer a client's technical question about metrics
- Document a formal campaign delivery
- Communicate a problem or performance drop to the client

## OPERATION MODES
- /quick → email/message template for a specific situation
- /standard → full delivery document or client report
- /deep → complete communication kit for client onboarding

## WHAT IT DOES
2 reference files:
**response-templates:** templates by type (campaign delivery, monthly report, problem communication, results presentation, upsell)
**escalation-guide:** when to escalate to human vs resolve with agent | urgency criteria | problem report format for the client

### Tone by situation
Positive result: objective, data-driven, celebrate without exaggeration
Negative result: honest, with identified cause and action plan already defined
Technical question: simple, no unnecessary jargon, with clear next steps

## SCOPE
ONLY for communication WITH agency's end clients.
Internal communication (Raphael ↔ agent) = direct, no templates.

## PROHIBITED
Generic templates without personalizing for the client and moment.
Sending a report with data without prior user approval.
Using for internal communication between user and agent.
Communicating a problem without an already identified cause and action plan.
